# MOODS - GDPR Compliance Documentation

**Last Updated: January 1, 2026**

## Overview

MOODS is committed to protecting the privacy and personal data of all users, including those in the European Union (EU) and European Economic Area (EEA). This document outlines how we comply with the General Data Protection Regulation (GDPR).

## 1. Legal Basis for Processing

### 1.1 Consent (Article 6(1)(a))
- Users provide explicit consent when creating an account
- Consent is freely given, specific, informed, and unambiguous
- Users can withdraw consent at any time

### 1.2 Contract Performance (Article 6(1)(b))
- Processing is necessary to provide the mood tracking service
- Data processing enables core app functionality

### 1.3 Legitimate Interest (Article 6(1)(f))
- Service improvement and analytics
- Security and fraud prevention
- Technical support and troubleshooting

## 4. User Rights Under GDPR

We fully support all GDPR rights:

### 4.1 Right to Access (Article 15)
- Users can view all their personal data in the app
- Data export available in standard formats (JSON, CSV)
- Response time: Within 30 days of request

### 4.2 Right to Rectification (Article 16)
- Users can edit their profile information
- Users can update mood entries
- Corrections processed immediately

### 4.3 Right to Erasure / "Right to be Forgotten" (Article 17)
- Full account deletion available in app settings
- All personal data permanently deleted within 30 days
- Backups purged within 90 days
- Users receive confirmation of deletion

### 4.4 Right to Data Portability (Article 20)
- Export data in machine-readable format (JSON)
- Export includes all mood data, notes, and settings
- One-click export feature in app

### 4.5 Right to Restrict Processing (Article 18)
- Users can pause data collection
- Account can be temporarily deactivated

### 4.6 Right to Object (Article 21)
- Users can object to data processing for marketing
- **Note:** MOODS does not use data for marketing purposes
- Users can opt-out of analytics

### 4.7 Rights Related to Automated Decision-Making (Article 22)
- MOODS does not use automated decision-making or profiling
- All insights are based on user's own data only
